# Suggested Google Sheet Schema

This starter kit does not connect to Google Sheets. The schema below is a planning reference for teams that later adapt the workflow into a private implementation.

## Leads

| Column | Purpose |
| --- | --- |
| lead_id | Stable local identifier for review and reporting. |
| business_name | Public business name. |
| industry | Segment or category used for review. |
| region | Market or service area. |
| website_url | Public website URL when available. |
| public_email | Public contact email if manually verified. |
| score | Lead score generated by local rules. |
| review_status | Suggested status: ready_to_review, hold, not_ready, excluded, duplicate. |
| notes | Human review notes and source context. |

## Outreach Review

| Column | Purpose |
| --- | --- |
| lead_id | Matches the Leads sheet. |
| reviewer | Person responsible for manual review. |
| approved_for_outreach | Yes/no decision before any private sending system is used. |
| opt_out_required | Reminder to include opt-out language. |
| last_reviewed_at | Review timestamp. |

## Activity Log

| Column | Purpose |
| --- | --- |
| activity_id | Stable activity identifier. |
| lead_id | Related lead. |
| activity_type | Example: scored, reviewed, dry_run_previewed, manually_contacted. |
| activity_at | Timestamp. |
| summary | Short human-readable description. |

Do not store API keys, OAuth tokens, cookies, or private credentials in a spreadsheet.
